# Branch Protection Rules

## 1. Required Status Checks
- Ensure that all required status checks pass before merging.

## 2. Require Pull Request Reviews
- All pull requests must be reviewed and approved by at least one other team member.

## 3. Restrict Who Can Push to Branch
- Only specific users or teams should be allowed to push to protected branches.

## 4. Dismiss Stale Pull Request Reviews
- Reviews should be dismissed if the pull request is updated after a review has been approved.

## 5. Require Linear History
- Merges should create a linear history to streamline the commit log.

## 6. Enforce Admins
- Even admins are subject to branch protection rules to ensure compliance.

## 7. Allow Force Pushes
- Disallow force pushes to protected branches to prevent unintended data loss.
